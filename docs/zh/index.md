# AION

AION 是自进化代码引擎——终结技术债，让代码库进入永生状态。

**Code Once, Live Forever. 一次编写，永远鲜活。**

它不是把每个文件孤立地看待，而是先从现有仓库中提取轻量级上下文，再用
`semgrep` 做第一轮规则扫描，只有在存在明确风险信号或明显上下文偏差时，
才交给 LLM 做进一步解释和判断。

## 这个项目解决什么问题

AI 生成代码经常“局部看起来合理”，但会悄悄偏离项目原有约定，从而引入安全风险：

- 整个项目都用 ORM，会话管理也统一，但新文件直接写了 `sqlite3`
- 大部分接口都有鉴权 decorator，新增 handler 却漏掉了
- 项目原本从环境变量读取密钥，生成代码却把密钥硬编码进文件

AION 要抓的就是这种“生成代码”和“项目上下文”之间的错位。

## 核心能力

- 在命令行扫描 Python 文件或整个仓库
- 通过文件标记、Git 历史或显式参数识别 AI 生成文件
- 基于静态分析提取项目上下文
- 先运行 `semgrep --config p/python`
- 再用 LLM 在仓库上下文里解释安全问题
- 输出 context gap、修复建议和 JSON 结果
