# 工作原理

AION 是一个分阶段执行的自进化分析流程。

## 1. 识别目标文件

CLI 会从输入路径中解析 Python 文件，并自动排除 `.git`、`.venv`、
`node_modules`、`__pycache__` 等常见目录。

然后再通过以下方式缩小到更可能是 AI 生成的文件：

- 文件内容里的生成标记，例如 `Generated by`
- Git 历史中的 Copilot / Cursor 迹象
- `--ai-generated` 显式指定的文件或目录

如果没有发现任何标记，它会给出 warning，然后扫描全部 Python 文件。

## 2. 提取仓库上下文

上下文提取器会基于静态分析构建一个轻量级仓库画像，包括：

- imports
- 鉴权 decorators
- 数据库访问模式
- 函数名
- 常见 ORM 和 HTTP client

这一步决定了后续能不能给出有价值的 context gap 判断。

## 3. 先跑 Semgrep

如果系统中安装了 `semgrep`，AION 会先运行 `p/python` 规则集，
把它作为低成本的第一轮筛查。

## 4. 只在必要时升级到 LLM

只有在以下情况，才会调用 LLM：

- `semgrep` 命中了问题
- 存在仓库上下文上的 fallback 风险信号
- 本地没有安装 `semgrep`

这样可以比“所有文件都发给模型”更聚焦，也更便宜。

## 5. 输出结果

每条问题都会尽量包含：

- 问题描述
- 严重级别
- 行号
- context gap
- 修复建议

结果支持富文本终端输出，也支持 JSON 输出。
